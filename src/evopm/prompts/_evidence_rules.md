## 证据引用规则（公共约定，所有 Agent 必须遵守）

1. **id 格式固定**：信号 `sig-001`、问题簇 `clu-01`、竞品发现 `cf-01`、技术发现 `tf-01`、需求 `req-01`、任务 `task-01`、历史需求 `ex-01`。
2. **只能引用提供给你的 id**：`evidence_refs` / `signal_ids` / `duplicate_of` / `duplicate_of_existing` 中出现的每一个 id，都必须来自本次输入里实际给出的对象；**严禁编造或臆测不存在的 id**。代码会在你输出后做闭包校验，非法引用一律剔除并记入审查输入，损害结论可信度。
3. **不允许编造来源**：`source_url` 只能填真实搜索结果 URL，或降级时填 `mock://<文件名>`；没有来源就留空，不要虚构链接。
4. **由你生成的 id 必须自洽**：在单次回答内为 clu/cf/tf/req/task 分配 id 时，保持格式正确、本次响应内唯一、从 01 顺序递增。
5. **结论与证据对应**：每条结论尽量绑定支撑它的上游 id；证据不足时如实标注（如 `evidence_strength=weak`），宁可标不确定也不要过度推断。
