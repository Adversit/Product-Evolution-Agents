# Dify — 上传 / 解析 / 引用能力（竞品材料）

Dify 是一站式 LLMOps 平台，知识库（Knowledge）模块支持上传 PDF、Word、Markdown、TXT 等文档，也支持从 Notion、网页同步。文档处理上提供「自动分段」与「自定义分段」两种模式，用户可设置分隔符、最大分段长度与重叠，并在创建知识库时**预览分段结果**，这一点对调参很友好。

解析状态方面，Dify 的文档列表会显示「处理中 / 可用 / 错误」等状态，长文档批量处理时有进度提示，失败的文档会标红并可重新处理；但其文档解析深度偏轻量，对复杂 PDF 版面、扫描件、表格的还原能力不及专门的深度解析引擎，复杂表格容易丢结构。

引用方面，Dify 在问答回答中支持「引用与归属」，可展示命中的分段及其来源文档，便于溯源，但定位粒度通常到分段级，不做原文像素级高亮。

已知问题与社区反馈：超大文件或海量文档批处理时偶发处理缓慢、卡在处理中；高质量索引（Q&A 模式、父子分段）会显著增加 token 与时间成本。

对当前需求的意味：分段预览、清晰的处理状态与失败重试是其相对成熟的能力，值得对标（adopt 方向）；而深度解析与表格还原不是其强项，不构成对 RAGFlow 的威胁。
